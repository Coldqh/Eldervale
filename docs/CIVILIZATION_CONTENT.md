# Цивилизации и расширяемый контент

## Границы системы

- **Цивилизация** владеет долгоживущими знаниями, эпохой, известными ресурсами и рецептами.
- **Государство** владеет властью, армией, законами, казной и территорией.
- **Поселение** владеет физическими зданиями, населением, производством и локальной экономикой.
- Государство может распасться или разделиться, но государства общей культуры продолжают использовать одну цивилизацию.
- Производство не хранит отдельную копию дерева технологий: доступность рецепта всегда вычисляется из цивилизации поселения.

## Поток данных

1. `src/content/coreContent.ts` собирает подключённые контент-пакеты.
2. `src/content/catalog.ts` проверяет ID, ссылки и циклы, затем создаёт индексированный каталог.
3. `src/sim/materialEconomy.ts` создаёт реальные ресурсы и рецепты из каталога.
4. `src/sim/civilizationSystem.ts` связывает культуры, государства и поселения, считает развитие и открывает технологии.
5. Производственные заведения получают только рецепты, разрешённые цивилизацией их поселения.
6. Сохранение хранит стабильные ID технологий и ключи рецептов, а не ссылки на позицию в массиве.

## Как добавить контент

Создай отдельный `CivilizationContentPack` и подключи его в список пакетов внутри `coreContent.ts`.

```ts
export const MY_PACK: CivilizationContentPack = {
  id: 'my-pack',
  version: 1,
  eras: [],
  technologies: [],
  resources: [],
  recipes: [],
};
```

### Новая технология

Укажи стабильный `id`, эпоху, предпосылки, стоимость, требования и открытия. Не используй название как ID. Удаление или переименование опубликованного ID требует миграции сохранений.

### Новый ресурс

Укажи стабильный `id` предмета и физические свойства. Рецепты ссылаются на ресурс через `templateId`. Технологически закрытый ресурс должен иметь `requiredTechnologyId`.

### Новый рецепт

Укажи стабильный `key`, входы, выходы, подходящие заведения и `requiredTechnologyId`. После открытия технологии рецепт автоматически попадёт во все совместимые заведения цивилизации.

### Новая эпоха

Эпоха задаёт не календарный год, а условия общества: обязательные технологии, минимальное население, урбанизацию и грамотность. Переход происходит только после выполнения всех условий.

## Инварианты

- ID пакетов, эпох, технологий, ресурсов и ключи рецептов уникальны.
- Все ссылки указывают на существующий объект каталога.
- Дерево технологий не содержит циклов.
- Освоенная технология всегда имеет освоенные предпосылки.
- Поселение использует цивилизацию своего государства.
- Заведение не может производить рецепт без требуемой технологии.
- UI не является источником истины для открытий и производства.

## Расширение в следующих версиях

Архитектура допускает отдельные системы распространения знаний, утраты технологий, заимствований между цивилизациями, колонизации и основания новых поселений. Эти механики должны менять состояние через `civilizationSystem`, а не напрямую редактировать списки рецептов в UI или экономике.
