# Feature Spec 031 — Premium Editorial Interface

## Проблема

Игровые системы уже образуют глубокую алкогольную экосистему, но интерфейс выглядит как набор однотипных служебных панелей. Контент, заведения, партии и коммерческие объекты не имеют достаточной визуальной иерархии.

## Пользователь и сценарий

Игрок управляет производством, брендами, продажами и активами с телефона и ПК. Он должен быстро видеть главное решение, находить нужный объект и открывать детали без перегруженного экрана.

## Наблюдаемое поведение

- на ПК используется постоянная боковая навигация и широкая контентная область;
- на телефоне сохраняется нижняя навигация и одна основная колонка;
- ключевые разделы получают собственную редакционную композицию и тематический визуал;
- списки сравнимых объектов используют ровные адаптивные сетки;
- поиск и фильтры расположены рядом с изменяемым контентом;
- подробности остаются в существующих модальных сценариях;
- игровая логика и доменные переходы не меняются.

## Критерии приёмки

- [x] единые токены цвета, типографики, размеров, интервалов, радиусов, теней и контролов;
- [x] отдельные desktop и mobile навигационные режимы;
- [x] premium wine / cocktail bar визуальное направление без градиентов и glassmorphism;
- [x] loading skeleton, error boundary, empty, disabled, selected, hover, active и success состояния;
- [x] длинные названия обрезаются или переносятся без горизонтального overflow;
- [x] реальные данные используются вместо UI-заглушек;
- [x] prefers-reduced-motion поддержан;
- [x] существующие обработчики действий и состояние сохранены.

## Негативные сценарии

- отсутствие изображений не скрывает текст и действие;
- пустые выборки показывают объяснение и дальнейший шаг;
- ошибка React-дерева не уничтожает локальное сохранение;
- узкий экран не требует hover и не получает горизонтальную прокрутку страницы.

## Данные и состояние

Новых доменных данных нет. Визуальные компоненты получают уже вычисленные показатели через существующие props.

## UX

Почти чёрный фон, винный акцент, шампань только для фокуса и редких premium-деталей. Иллюстрации созданы как локальные SVG и не требуют сети.

## Производительность

Визуалы — компактные SVG. Новых runtime-зависимостей нет. Анимации ограничены короткими opacity/transform-переходами.

## Миграция и откат

Миграция сохранений не требуется. Откат выполняется заменой UI-файлов; schemaVersion не меняется.

## Не входит в задачу

Изменение экономики, симуляции, рецептов, AI компаний и доменных контрактов.
