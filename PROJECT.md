# PROJECT.md — память Eldervale

## Название и цель

- Название: Eldervale.
- Жанр: автономная процедурная фэнтези-экосистема и игра внутри неё.
- Основной результат: мир существует, развивается и ломается по единым законам независимо от игрока.
- Приоритет: сначала физически и причинно честный мир, затем игровые действия и интерфейс.

## Текущий статус

- Версия: 6.6.0.
- Работает: прожитая история, персонажи и семьи, поселения, физические здания и интерьеры, предприятия, товары, торговля, месторождения, технологии, институты, государства, армии, климат, экология и сохранения.
- Текущий вертикальный срез: единая физическая финансовая экономика.
- Исправлено в 6.6.0: продажи, зарплаты, налоги, государственные расходы и долги проходят через реальные счета и журнал проводок.
- Главный технический долг: часть старых денежных операций вне основного хозяйственного цикла ещё требует перевода на явные проводки; ускоренное время и обычный ход требуют доказательства эквивалентности.
- Ближайшая цель: убрать готовые королевства из первого года и вывести государство из реальных политических общин.

## Стек

- TypeScript 7, React 19, Vite 8.
- Клиентское приложение без обязательного сервера.
- Сохранения в браузере с JSON-экспортом.
- Детерминированные smoke-сценарии для доменных систем.
- GitHub Pages и GitHub Actions.

## Архитектура

- `WorldState` хранит состояние мира.
- `src/sim` содержит доменные системы и единый месячный pipeline.
- Физические `WorldItem`, `Household`, `Building`, `Establishment`, армия и персонаж являются источниками истины.
- Агрегаты поселения допустимы только как вычисляемые проекции.
- UI читает мир и вызывает доменные переходы, но не создаёт отдельные правила.

## Инварианты

- Материя не появляется и не исчезает без события производства, потребления, порчи, утраты или перемещения.
- Один персонаж не может одновременно находиться в несовместимых состояниях.
- Экспедиция состоит из существующих живых людей и семей и несёт физический груз.
- Ускорение времени не должно менять итоговые законы мира.
- История описывает изменение состояния, а не заменяет его.
